# javaTest
Java练习、测试的项目

## 📂 项目结构说明

在 Spring Boot 开发中，我们采用经典的分层架构。以下是核心文件夹的含义与职责，把后端开发想象成经营一家餐厅：

### 1. `entity` (实体层) —— 📋 菜单/食材定义
*   **作用**：定义数据结构，与数据库表一一对应。
*   **类比**：**菜单或食材清单**。
    *   告诉程序：“咱们店里有‘用户’(User)、‘商品’(Product) 这种东西，它长什么样（有哪些属性）。”
*   **示例**：
    *   `User.java` -> 对应数据库 `users` 表
    *   `Product.java` -> 对应数据库 `products` 表

### 2. `mapper` (映射层) —— 📦 仓库管理员 (原 repository)
*   **作用**：负责数据的增删改查，是唯一能直接操作数据库（仓库）的接口。
*   **别名**：在 JPA 中叫 `Repository`，在 MyBatis 中叫 `Mapper` (或 DAO)。
*   **类比**：**仓库管理员**。
*   **职责**：
    *   继承自 `BaseMapper` (MyBatis-Plus)，自带增删改查技能。
    *   如果需要复杂 SQL，也可以在这里手写 SQL 语句。
*   **示例**：
    *   `UserMapper.java` -> 负责管理 `User` 数据的存取

### 3. `config` (配置层) —— 👔 餐厅经理/装修队
*   **作用**：存放项目启动时的全局配置、初始化逻辑或安全规则。
*   **类比**：**经理制定的规则**。
    *   不负责具体的炒菜（业务），而是负责环境维护。
    *   比如：“开业时先检查有没有食材（数据初始化）”、“允许谁进门（跨域/安全配置）”。
*   **示例**：
    *   `DataInitializer.java` -> 项目启动时自动初始化数据
    *   `CorsConfig.java` -> 解决前后端跨域问题

## ❓ 常见疑问解答

### Q1: `Controller` 和 `Mapper` 既然都是接口，为什么分开放？
这就好比**前台服务员**和**后厨库管**的区别：
*   **Controller (AuthController)** = **前台服务员** 💁‍♂️
    *   **面向客人 (前端)**：负责接待 HTTP 请求，解析 JSON，校验参数。
    *   它不懂数据库，只懂怎么和人打交道。
*   **Mapper (UserMapper)** = **后厨库管** 📦
    *   **面向仓库 (数据库)**：负责执行 SQL，搬运数据。
    *   它不懂 HTTP，只懂怎么和数据库说话。
*   **如果不分开**：就像让满身灰尘的库管直接去大堂接待客人，既不安全（直接暴露数据库），分工也混乱。

### Q2: `Service` (业务层) 是什么角色？
*   **角色**：**厨师** 👨‍🍳。
*   **职责**：它位于 Controller 和 Repository 中间。
    *   Controller 把客人的需求告诉 Service。
    *   Service 指挥 Repository 拿食材，然后进行加工（复杂的业务逻辑计算，比如计算折扣、发送邮件、调用第三方 API）。
    *   加工好后交给 Controller 端出去。
*   **对比 ES/Radius**：
    *   Service 这里的概念是指**代码逻辑层**（Java 类），不是指外部服务（如 Elasticsearch 或 Radius 服务器）。
    *   但在 Service 的代码里，确实可能会去调用 ES 或 Radius 这种外部设施。

### Q3: 文件夹名字是硬性规定吗？
*   **不是硬性规定，但 是约定俗成的行业标准**。
*   你可以叫 `dao` 而不是 `repository`，也可以叫 `model` 而不是 `entity`。
*   但是！为了让其他开发者（或者未来的你自己）能一眼看懂，**强烈建议遵守标准命名**。

### Q4: 我可以把 Controller 放到单独的文件夹里吗？
*   **当然可以，而且强烈推荐！**
*   当项目变大时，把所有 Controller 丢在根目录下会很乱。
*   **做法**：创建一个 `controller` 包，把 `AuthController`、`HelloController` 移进去。Spring Boot 会自动扫描到它们，不用改任何配置。
